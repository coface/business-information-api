Data APIs is a service offered by ICON, Coface Business Information offer. It allows you to request datas about a company, a country or a sector to help you to assess and manage your risks on large portfolios.

Products

This API allows you to order and monitor :

  • Credit Opinion (simple or advanced)
  • Debtor risk assessment (with or without history and probability of default)
  • Company financial data
  • Country risk assessment
  • Sector risk assessment

The description of each product can be found here.

Use cases

I want to know the country risk of trading with Argentina

  • Check if the Country Risk Assessment (CRA) is available for this country by calling GET /assessments
  • If yes, order the data via POST /publications/orders
  • Retrieve the publication by searching it with the deliveryId provided in the previous step

I want to know the risk of trading with automobile sector in Germany

  • Check if the Sector Risk Assessmentv(SRA) is available for this country by calling GET /assessments
  • If yes, order the data via POST /publications/orders
  • Retrieve the publication by searching it with the deliveryId provided in the previous step

I want to know the risk of trading with a company

  • Check if the Debtor Risk Assessment (DRA) is available for this country by calling GET /assessments
  • If yes, search the company through GET /company
    • If you find the company, copy its identifier
    • If you don't find it, ask for an identification thank to POST /companies/identifications, copy the response id
  • Order the data via POST /publications/orders
  • Retrieve the publication by searching it with the deliveryId provided in the previous step

I want to know the Coface credit opinion on a company

  • Check if the Credit Opinion (CO) is available for this country by calling GET /assessments
  • If yes, search the company through GET /company
    • If you find the company, copy its identifier
    • If you don't find it, ask for an identification thank to POST /companies/identifications, copy the response id
  • Order the data via POST /publications/orders
  • Retrieve the publication by searching it with the deliveryId provided in the previous step

Version

We keep on doing our best to minimize impacts on our customers but we will sometimes need to modify our interface contract in order to enhance our features.

Such modifications will bring the need of a new version. Each version will be classified as a minor or a major one.

When released, a minor version has no impact on the consumer. Some bugs have been fixed and some evolutions have been given. It will sometimes bring a modification of the interface contract but being an upgrade i.e. new fields and/or endpoints, the consumer is free to decide whether he wants to add the potential new features or not.

However, a major version will impact the consumer in any case: both in the url (where the version stands) and in its code. New developments will be needed to be able to benefit from the modifications.

Coface will maintain at most two versions of the API.

Security

Security is a priority for Coface. Security Audit with penetration tests have been performed for 2020 leading to a green light.

Coface uses Oauth2 as authentication mechanism; every call must include a JWT token as well as an API key:

  • Your JWT token can be generated thanks to the credentials that have been provided to you for accessing your private account. It is issued by a secured identity provider (AWS Cognito).
  • Your API key can be found directly in your private account.

Please note that your API key is unique to each user-application pair. It contains your privileges so be sure to keep it secret ! Do not share it publicly in such areas as GitHub, client-side code, and so forth.

All API requests must be made over HTTPS. Calls made over plain HTTP will fail. API requests without authentication will also fail.

General principles

Language

For content which is language-specific, the language can be selected in the user's profile.

Alternatively the "Accept-Language" header may be used to specify a language, which will then override the language selected in the user's profile.

Pagination

To avoid performance issue, we have limited all our searches to a max number of elements.
This limitation is specified in the response header "Accept-Range".
Accept-Range : ${ressource} ${max)

The range query parameter allow you to request specifics items. It's composed like this {offset} - {nb of item to retrieve}.
For example, to get the ten first item, range = 0 - 10. For the 5 next items 10 - 5.
Be aware that there is a maximal limitation for the number of item to retrieve. If it's 20 for example, 0 - 50 will return only 20 items.
The number of items that matched the criteria must be found in the response header "Content-Range".
