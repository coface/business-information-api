Report APIs is a service offered by ICON, Coface Business Information offer. It allows you to request informations about a company.

Products

This API allows you to order and monitor :

  • Business report (full or snapshot)
  • Credit report
  • Debtor risk assessment (with or without history and probability of default)

The description of each product can be found here.

Note : Specific products can be available depending on your contrat with ICON.

Access

Before calling our API, you need to

  1. Be registred in our customer database. You can get it by contacting us.
  2. Create an account on our dev portal.
  3. Subscribe on "API Coface - Authentication".
  4. Subscribe on "ICON Report API".

Security

Our API uses an OAuth2 protocol. You will need to request a token available 1H, before calling our API. (See Authentication process)

Process

Authentication

To get the token, you must call the /token endpoint of the "API Coface - Authentication" with your credential : login and password of your dev portal account in the request body and your api key in the "x-api-key" request header.

The received token must be put in the "authorization" request header along with the "x-api-key" request header.

Be aware that this token is valid only 1h.

Technical documentation can be found on dev portal.

Company search

To order monitoring or report, you need to find on which company you want the data.

To identify a company, you have two possibilities : use the Coface Id of the company or use an unique id from a national referential.

For example : in France, all companies must be registred in the National Institute of Statistics and Economic Studies (INSEE). This repository is used by Coface.

Version

We keep on doing our best to minimize impacts on our customers but we will sometimes need to modify our interface contract in order to enhance our features.

Such modifications will bring the need of a new version. Each version will be classified as a minor or a major one.

When released, a minor version has no impact on the consumer. Some bugs have been fixed and some evolutions have been given. It will sometimes bring a modification of the interface contract but being an upgrade i.e. new fields and/or endpoints, the consumer is free to decide whether he wants to add the potential new features or not.

However, a major version will impact the consumer in any case: both in the url (where the version stands) and in its code. New developments will be needed to be able to benefit from the modifications.

Coface will maintain at most two versions of the API.

General principles

Language

For content which is language-specific, the language can be selected in the user's profile.

Alternatively the "Accept-Language" header may be used to specify a language, which will then override the language selected in the user's profile.

Pagination

To avoid performance issue, we have limited all our searches to a max number of elements.
This limitation is specified in the response header "Accept-Range".
Accept-Range : ${ressource} ${max)

The range query parameter allow you to request specifics items. It's composed like this {offset} - {nb of item to retrieve}.
For example, to get the ten first item, range = 0 - 10. For the 5 next items 10 - 5.
Be aware that there is a maximal limitation for the number of item to retrieve. If it's 20 for example, 0 - 50 will return only 20 items.
The number of items that matched the criteria must be found in the response header "Content-Range".
Content-Range : ${offset}–${limit}/${count}

Rate Limits

API access rate limits are applied at a per-key basis in unit time. Access to the API using a key is limited to 60 requests per minute. In addition, every API response is accompanied by the following set of headers to identify the status of your consumption.

HeaderDescription
X-RateLimit-LimitThe maximum number of requests that the consumer is permitted to make per minute.
X-RateLimit-RemainingThe number of requests remaining in the current rate limit window.

If you are hitting this limit, further requests will be rejected with an HTTP 429 error. If you believe that due to the nature you intend to use the api your limit needs to be increased then please contact us.
